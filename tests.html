Mes Listes — tests

Ces tests chargent la véritable application dans un cadre invisible et exercent son code réel — pas une copie qui pourrait diverger.

Tes listes sont mises à l'abri. Les tests ont besoin d'écrire dans le stockage de l'app. Son contenu est sauvegardé avant de commencer et remis à l'identique à la fin, même si un test échoue. Rien ne démarre tant que tu n'as pas appuyé sur le bouton.
Ouvre cette page depuis le serveur local. Là, elle a été ouverte directement depuis le disque. Le navigateur traite alors chaque fichier comme un site étranger aux autres, et interdit à cette page de lire dans l'application — les tests ne peuvent pas s'exécuter.

Double-clique sur Lancer les tests.bat dans le dossier de l'application : il démarre le serveur et rouvre cette page au bon endroit.

Hors de portée de ces tests

  • Tout ce qui demande un vrai compte Firebase : connexion, règles d'accès, synchronisation réelle entre appareils.
  • La feuille de partage iOS, donc la sauvegarde vers iCloud.
  • Les notifications, et le comportement du service worker hors connexion.
  • Le rendu visuel : ces tests vérifient la structure et les valeurs, pas l'apparence.
